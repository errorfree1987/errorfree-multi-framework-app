# Error-Free® 企業試用 — 架構與資料安全說明

> **用途**：給千人以上大公司主管試用時，說明基本架構與資料安全，化解「上傳文件、資料外洩」疑慮  
> **適用對象**：企業 IT 主管、法務、決策者  
> **語言**：中文（對外說明可改為英文）

---

## 一、基本架構（一句話版）

**Error-Free® 是多租戶 SaaS：企業員工登入 Portal → 選擇 Framework → 進入 Analyzer 上傳文件 → AI 輔助審閱後下載報告。每位企業的資料完全隔離，彼此看不到。**

## 三、資料安全 — 如何化解「怕資料外洩」

### 3.1 企業最常問的三個問題

| 問題 | 簡短回答 |
|------|----------|
| **你們會保存我們上傳的文件嗎？** | 會。為提供分析與報告，需暫存於我們後端。但資料與其他企業完全隔離，且可依貴司要求設定保留天數或刪除。 |
| **會不會被其他公司看到？** | 不會。採用租戶隔離（tenant isolation），貴司的資料僅貴司授權成員可見。 |
| **會不會拿我們的資料去訓練 AI 或外洩？** | 不會。我們不將客戶資料用於訓練模型，亦不與第三方共享。 |

---

### 3.2 六點資料安全承諾（可做成一頁簡報）

#### 1. 租戶隔離（Tenant Isolation）

- 每間企業為獨立「租戶」
- 資料庫依 `tenant_id` 嚴格區隔
- **A 公司成員無法存取 B 公司的任何資料**

#### 2. 存取控制（Access Control）

- 僅經授權的成員（依 email 登入）可進入該企業的 Analyzer
- 成員列表由 Admin 管理，可隨時停用或刪除帳號

#### 3. 傳輸加密（Encryption in Transit）

- 全站 HTTPS
- 上傳、下載皆經加密連線

#### 4. 儲存安全（Storage）

- 資料存放於 Supabase（可選 region，如新加坡、美西）
- 依雲端供應商標準加密儲存

#### 5. 不訓練、不共享（No Training, No Sharing）

- **不**將客戶上傳文件用於訓練任何 AI 模型
- **不**與第三方共享或出售客戶資料
- 僅用於提供貴司訂閱的服務

#### 6. BYOK 選項（未來）

- Phase C 將支援企業自帶 API Key（BYOK）
- 企業可選擇讓分析請求送往**自家**的 OpenAI / Azure OpenAI 端點
- 文件內容僅經貴司指定的 AI 服務，進一步降低資料疑慮

---

### 3.3 試用期間的具體做法（可寫進合約或同意書）

| 項目 | 說明 |
|------|------|
| **試用範圍** | 僅供評估，不涉及正式營運敏感資料 |
| **資料保留** | 試用結束後可依要求刪除該租戶所有資料 |
| **存取紀錄** | 登入、上傳、分析等操作可寫入 audit log，供稽核 |
| **聯絡窗口** | 若有資安疑慮，可直接聯繫 Error-Free 負責人 |

## 四、一句話總結（給主管背）

> **「貴司上傳的文件僅供 Error-Free 提供分析服務使用，與其他企業完全隔離，我們不訓練 AI、不共享資料。試用期滿可要求刪除。未來可選 BYOK，讓分析經由貴司自己的 AI 端點。」**

## 五、若對方追問技術細節

| 問法 | 建議回答 |
|------|----------|
| 資料存在哪裡？ | 存在 Supabase（PostgreSQL），可指定 region，例如亞太區。 |
| 誰能登入看我們的資料？ | 僅貴司 Admin 加入的成員；Error-Free 運維僅在必要時為技術支援存取後台，且受權限與 audit 紀錄約束。 |
| 有沒有通過 ISO 27001？ | （依實況填寫）目前／規劃中。 |
| 可以簽 NDA 嗎？ | 可以，可依貴司法務需求提供 NDA 範本。 |
